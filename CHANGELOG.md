# Changelog

All notable changes to this TypeScript project template will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2025-09-23

### Added

- Initial TypeScript project template
- TypeScript configuration with strict mode enabled
- ESLint configuration with TypeScript-specific rules
- Prettier configuration for consistent code formatting
- Jest testing framework with TypeScript support
- Husky pre-commit hooks for code quality
- lint-staged for running linters on staged files only
- VS Code workspace settings and recommended extensions
- Comprehensive project structure with examples
- Path mapping support with `@/` aliases
- npm scripts for common development tasks
- Sample utilities and types with full documentation
- Test examples demonstrating best practices
- Comprehensive README with setup and usage instructions
- MIT License
- Environment variables template
- Git hooks configuration

### Features

- **Development Tools**: TypeScript, ESLint, Prettier, Jest
- **Code Quality**: Pre-commit hooks, automated linting and formatting
- **VS Code Integration**: Optimized settings, recommended extensions, tasks
- **Testing**: Jest configuration with coverage reporting
- **Path Mapping**: Clean imports with `@/` prefix
- **Documentation**: Comprehensive README and inline code documentation

### Configuration Files

- `package.json` - Project dependencies and scripts
- `tsconfig.json` - TypeScript compiler configuration
- `.eslintrc.json` - ESLint rules and settings
- `.prettierrc` - Prettier formatting rules
- `jest.config.js` - Jest testing configuration
- `.gitignore` - Git ignore patterns
- `.vscode/` - VS Code workspace configuration
- `.husky/` - Git hooks configuration

### Example Code

- Calculator utility class with comprehensive methods
- Greeting utilities with error handling
- Helper functions for common operations
- TypeScript type definitions for User and common types
- Jest test examples for all utilities
- Main entry point demonstrating usage patterns
